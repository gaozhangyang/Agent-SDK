# Markdown 渲染问题修改意见

> 针对 `seq:001` / `seq:002` / `seq:004` 等 Agent 日志格式的渲染优化建议

---

## 问题一：代码块嵌套冲突（严重）

### 问题描述

`<details>` 折叠块内部使用三个反引号（` ``` `）包裹代码，当外层 Markdown 环境已有同级代码块时，解析器会提前闭合代码块，导致内容错乱或渲染失败。

### 错误示例

````markdown
<details><summary>完整 input / output</summary>
### output
```json
{ "name": "..." }
```
</details>
````

### 修改方案

内层代码块改用 **四个反引号**（` ```` `），与外层三个反引号区分：

`````markdown
<details><summary>完整 input / output</summary>

### output

````json
{
  "name": "Reinforcement Learning"
}
````

</details>
`````

## 问题二：表格内容过长，渲染混乱（严重）

### 问题描述

`content` / `result` 列塞入了大段长文本，表格会出现以下问题：

- 单元格宽度极度失衡
- 文字堆叠、溢出或换行混乱
- 难以快速扫读关键信息

### 错误示例

```markdown
| content | # [论文标题]  **ArXiv ID**: {arxiv_id} **发表日期**: ... |
```

### 修改方案

表格只保留关键元信息（路径、简短预览），完整内容移入 `<details>` 折叠块：

```markdown
| 字段    | 值                                      |
| ------- | --------------------------------------- |
| path    | `$research/.../paper_summary.md`        |
| preview | `# [论文标题] · ArXiv ID: {arxiv_id}…`  |
```

## 问题三：`<details>` 前后缺少空行（中等）

### 问题描述

Markdown 规范要求块级 HTML 元素（如 `<details>`）前后必须有**空行**，否则内部的标题、列表、代码块等将不会被正确渲染，直接显示为纯文本。

### 错误示例

```markdown
## seq:001
<details><summary>完整 input / output</summary>
### output
内容...
</details>
## seq:002
```

### 修改方案

`<details>` 标签前后各加一个空行，内部各块元素之间也保留空行：

```markdown
## seq:001

<details><summary>完整 input / output</summary>

### output

内容...

</details>

## seq:002
```

## 问题四：标题层级污染（中等）

### 问题描述

折叠块内部的模板内容包含 `# [论文标题]`（h1 级标题），会干扰页面整体的标题层级结构，影响侧边栏导航、锚点跳转等功能。

### 修改方案

**方案 A**：将折叠内容中的标题整体降级（`#` → `###`，`##` → `####`）

**方案 B**（推荐）：将模板内容整体放入代码块，明确表示"这是模板示例，非页面结构"：

````markdown
<details><summary>完整 output（模板内容）</summary>

````markdown
# [论文标题]
## 研究问题
...
````

</details>
````

## 问题五：seq 卡片结构不统一（建议）

### 问题描述

各 `seq` 模块结构松散，没有统一的视觉层次，阅读体验较差。

### 推荐统一结构

每个 seq 卡片按以下三层结构组织：

`````markdown
## `seq:001` · `10:40:58` · 📖 read

| 字段    | 值                                |
| ------- | --------------------------------- |
| path    | `$research/.../paper_summary.md`  |
| preview | `# [论文标题] · {arxiv_id}…`      |

<details><summary>完整 input / output</summary>

### output

````markdown
# [论文标题]
...
````

</details>
`````

**三层结构说明：**

1. **标题行** — seq 编号 + 时间戳 + 操作类型 emoji
2. **摘要表格** — 仅展示 path 和内容预览（不超过 80 字符）
3. **折叠详情** — 完整 input / output，内层代码块用四个反引号
